# OpenProblemAtlas

> 一个活的、机器可读的开放问题图谱——面向研究者、AI 智能体和定理证明器。

发现经过验证的开放问题，从文献中挖掘新线索，追踪可复现的 AI/人类尝试——全部汇聚在一个开放数据集中。

---

## 内容概览

| 层级 | 描述 | 数量 |
|------|------|------|
| **Atlas** | 经人工验证的开放问题，附带规范来源 | 50+ |
| **Radar** | 机器挖掘的候选问题，等待审核 | 持续增长 |
| **Lab** | 可复现的 AI/人类尝试记录 | 持续增长 |
| **Explorer** | 可搜索、可筛选的 Web 界面 | [即将上线] |

## 核心特性

- **已验证问题** — 每条记录都有规范陈述、来源、状态证据和审核记录
- **新鲜雷达线索** — 自动化管道从 arXiv、综述和问题集中挖掘新候选
- **形式化链接** — 筛选 Lean/Coq 形式化问题或 `formalization-wanted` 标签
- **求解器就绪标签** — 找到适合 SAT、SMT、计算搜索或 LLM 推理的问题
- **低关注度过滤** — 发现重要但尚未获得足够关注的问题
- **可复现尝试** — 追踪 AI 和人类尝试，带完整溯源（模型、提示词、工具、日期）
- **JSON / Parquet 快照** — 下载结构化数据用于自己的分析

## 领域范围（v1）

1. **数学** — 数论、组合数学、代数、分析、几何、拓扑
2. **理论计算机科学** — 复杂性理论、算法、图论、密码学
3. **数学物理** — 量子信息、统计力学、量子场论

## 数据模型

每个已验证问题包含：

- 规范标题和陈述
- 领域和子领域标签
- 状态（open / partially_solved / solved / disproved / conditional / independent / ambiguous）
- 带定位符的来源引用
- 状态证据
- 审核元数据（审核者、日期、置信度）
- 评分（影响力、低关注度、可工具化、形式化程度、AI 适配度）
- 形式化状态
- AI/工具就绪标签

详见 [docs/taxonomy.md](../taxonomy.md) 的完整分类法和 [schema/](../../schema/) 的 JSON Schema。

## 来源策略

我们区分三个层级的来源：

1. **一级（规范来源）：** 问题集、原始论文、综述、教科书、专家维护页面
2. **二级（辅助来源）：** 社区讨论、基准论文、权威数据库
3. **三级（仅限线索）：** 新闻、博客、社交媒体——永远不作为规范来源

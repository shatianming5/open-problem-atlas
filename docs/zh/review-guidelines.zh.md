# 审核指南

## 角色

### 维护者

- 可以批准任何 PR
- `open -> solved/disproved` 状态变更时必须参与
- 管理项目方向和范围

### 领域审核者

- 特定领域的专家（数学、理论计算机科学、物理学）
- 可以批准其领域内的新问题和状态变更
- 每个已验证问题至少需要一位领域审核者的批准

### 贡献者

- 提交 issue、PR 或数据的任何人

## 新问题审核清单

- [ ] `id` 遵循命名约定：`opa.<domain>.<subdomain>.<slug>`
- [ ] `title` 是规范的且无歧义的
- [ ] `statement.canonical` 是精确且自包含的
- [ ] `status.label` 正确且有证据支持
- [ ] 至少有一个带定位符的 Tier 1 来源
- [ ] 至少有一条状态证据条目
- [ ] `domain` 和 `subdomains` 来自已批准的分类体系
- [ ] `last_reviewed_at` 已设置
- [ ] 机器生成的字段（如有）带有 `disclaimer: unverified`
- [ ] 非现有条目的重复项（检查别名）
- [ ] 所有引用的 `source_id` 值存在
- [ ] 所有关系引用指向有效的问题 ID

## 状态变更审核清单

- [ ] 提供了主要证据（论文链接、DOI、证明引用）
- [ ] 添加了状态历史记录
- [ ] 至少一位领域审核者批准
- [ ] 对于 `open -> solved/disproved`：需要维护者批准
- [ ] 保留原始条目（状态演进，而非删除）

## 尝试审核清单

- [ ] `problem_id` 引用了有效的已验证问题
- [ ] 指定了 `actor` 或带版本的 `model`
- [ ] 存在方法描述或提示词摘要
- [ ] 记录了工具和环境
- [ ] 标记为 `verified` 或 `unverified`
- [ ] 未经验证的声明未混入问题的真实数据层

## 线索审核清单

- [ ] 至少有一个来源引用
- [ ] 包含粗略的问题描述
- [ ] 非明显的重复项
- [ ] 提供了领域分类

## 常见错误注意事项

1. **混淆"未来工作"与"开放问题"** -- 论文中的未来工作建议不一定是定义明确的开放问题
2. **过时的状态** -- 2010 年综述中列为开放的问题可能此后已被解决
3. **缺乏精确性** -- "X 是否成立？"但没有精确定义 X
4. **别名混淆** -- 同一个问题以不同名称出现
5. **机器字段中的 AI 幻觉** -- LLM 生成的摘要凭空捏造结果

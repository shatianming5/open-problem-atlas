# 分类法

## 记录类型

图谱中的每个条目严格属于以下三种类型之一：

### 1. 问题（`problem`）
经过验证的开放问题，具有精确的规范陈述、至少一个一级来源、确认的状态和人工审核记录。

### 2. 线索（`lead`）
尚未完全验证的候选开放问题。可能不精确、重复或已解决。预期有噪声。

### 3. 尝试（`attempt`）
针对某个已验证问题的 AI 或人类尝试记录。永远不进入问题的真相层。

## 领域（v1）

### 数学（`mathematics`）
子领域：数论、组合数学、图论、代数、分析、几何、拓扑、概率、动力系统、逻辑与基础

### 理论计算机科学（`theoretical-cs`）
子领域：复杂性理论、算法、图算法、密码学、信息论、计算几何、形式语言、分布式计算、量子计算

### 数学物理（`mathematical-physics`）
子领域：量子信息、统计力学、量子场论、广义相对论、弦理论、凝聚态理论

## 状态标签

| 状态 | 含义 |
|------|------|
| `open` | 无已知解法或反证 |
| `partially_solved` | 有部分进展但未完全解决 |
| `solved` | 存在完整解法且有证据 |
| `disproved` | 存在反例或反证 |
| `conditional` | 在未证明假设下已解决 |
| `independent` | 已证明独立于标准公理 |
| `ambiguous` | 陈述有争议或不明确 |
| `retired_duplicate` | 已合并到另一条目 |

## 评分（0.0–1.0）

| 评分 | 含义 |
|------|------|
| `impact` | 问题对其领域的重要性 |
| `underexplored` | 近期受到的关注度多低 |
| `toolability` | 适合计算工具的程度 |
| `formality` | 陈述可形式化的精确程度 |
| `ai_fit` | 适合 AI 方法的程度 |

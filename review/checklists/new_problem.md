# New Problem Review Checklist

## Identity
- [ ] ID follows `opa.<domain>.<subdomain>.<slug>` convention
- [ ] Title is canonical and unambiguous
- [ ] Aliases include common alternative names

## Statement
- [ ] Canonical statement is precise and self-contained
- [ ] A mathematician can understand the problem from the statement alone
- [ ] Informal statement is accessible to non-specialists

## Status
- [ ] Status label is correct (open/solved/etc.)
- [ ] Status evidence is provided
- [ ] Confidence level is appropriate
- [ ] `last_reviewed_at` is set to a recent date

## Sources
- [ ] At least one Tier 1 canonical source
- [ ] Source has title, year, and URL/DOI
- [ ] Locator pinpoints where the problem appears
- [ ] Status evidence references are valid

## Classification
- [ ] Domain is correct (mathematics/theoretical-cs/mathematical-physics)
- [ ] Subdomains are appropriate
- [ ] Problem type is correct
- [ ] Answer type makes sense

## Quality
- [ ] Not a duplicate of existing entry
- [ ] Machine-generated fields have `disclaimer: unverified`
- [ ] Scores are reasonable
- [ ] Partial results are real and cited
